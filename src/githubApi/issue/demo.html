前端页面性能优化的指标通常包括以下几个方面：

  1. 加载时间（Load Time）：页面从请求到完全加载所需的时间。通常用页面的首次加载时间（First Contentful Paint, FCP）或完全加载时间（Load Event End）来衡量。

  2. 渲染性能（Rendering Performance）：页面渲染的速度和流畅度。通常用关键渲染路径（Critical Rendering Path）和渲染帧率（Frame Rate）来衡量。

  3. 交互性能（Interactivity）：用户与页面交互的反应速度和体验。通常用首次输入延迟（First Input Delay, FID）和交互响应时间（Time to Interactive, TTI）来衡量。

  4. 可访问性（Accessibility）：页面对于残障人士的可访问性。通常用无障碍标准（Web Content Accessibility Guidelines, WCAG）来衡量。

这些性能指标可以通过各种工具和技术来量化和优化。例如，可以使用浏览器的开发者工具（DevTools）来分析页面的加载和渲染性能，并进行性能优化；可以使用代码压缩、资源合并、CDN 加速等技术来优化页面的加载速度；可以使用动画优化、响应式设计、事件委托等技术来提高页面的渲染性能和交互性能；可以使用无障碍标准和工具来确保页面的可访问性。

总之，前端页面性能优化需要综合考虑各个方面的性能指标，并使用合适的工具和技术来量化和优化页面的性能。
